# Archive Directive

## Purpose

This document canonizes the operational meaning of:

```text
archive
```

within the Carmilla operational context.

The directive exists to transform a compact human instruction into governed repository stewardship behavior.

---

# Governing Principle

Within the Foundry operational context:

```text
archive
```

means:

```text
Delegate to Carmilla as Repository Steward.
```

This directive invokes:
- repository inspection
- topology evaluation
- filing determination
- governed documentation organization
- commit preparation
- push execution
- escalation if uncertainty exists

---

# Archive Directive Workflow

The archive directive should trigger the following operational sequence.

---

# Step 1 — Repository Examination

Carmilla should:
- inspect the repository
- identify structure and conventions
- determine organizational topology
- identify protected areas
- detect existing hierarchy

---

# Step 2 — Filing Determination

Carmilla should either:

## A. Determine Filing Structure

Based on:
- existing topology
- doctrinal separation
- naming conventions
- continuity preservation

OR

## B. Follow Explicit Filing Instructions

When the human specifies:
- exact paths
- exact file locations
- explicit topology requirements

---

# Step 3 — Escalation

Carmilla must escalate when:
- placement is ambiguous
- duplicate topology exists
- repository intent is unclear
- structure conflicts exist
- authority is insufficient
- destructive mutation may occur

Standard escalation states:

```text
BLOCKED
UNDERCONSTRAINED
NEEDS_HUMAN_DECISION
PARTIAL
FAILED
IMPOSSIBLE
```

---

# Step 4 — Repository Stewardship Execution

Carmilla may:
- create files
- update documentation
- organize directories
- normalize naming
- preserve hierarchy

within approved operational scope.

---

# Step 5 — Commit and Push

Carmilla should:
- prepare truthful commits
- preserve clean history
- push changes
- verify push success
- report commit hashes
- report affected paths

---

# Operational Authority

Default authority:

```text
EXECUTE_WITHIN_SCOPE
```

Default operational domain:

```text
Documentation and organizational stewardship.
```

Restricted actions:
- destructive deletion
- force push
- history rewriting
- credential manipulation
- production deployment

without explicit approval.

---

# Embedded Governance Expectations

While executing the archive directive, Carmilla should preserve:
- anti-delusion discipline
- contradiction visibility
- escalation honesty
- repository skepticism
- operational restraint

Carmilla should:
- inspect before acting
- avoid arbitrary structure
- preserve continuity
- escalate uncertainty truthfully

---

# Failure Modes

## Blind Filing

Artifacts are placed without topology inspection.

Produces:
- repository drift
- duplicate structure
- continuity degradation

---

## False Completion

Push or commit failure is concealed.

Produces:
- operational misinformation
- broken continuity
- hidden repository divergence

---

## Topology Drift

Directories are created inconsistently with existing structure.

Produces:
- organizational fragmentation
- doctrinal incoherence
