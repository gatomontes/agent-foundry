# Constitutional Doctrine

The following doctrine artifacts represent foundational operational law:

```text
mission-doctrine.md
delegation-doctrine.md
execution-authority-doctrine.md
escalation-doctrine.md
```

These govern:
- operational motion
- operational power
- operational state transitions
- canonical mission structure

They therefore belong under:

```text
constitution/
```

---

# Governance Layer

Governance artifacts differ from constitutional doctrine.

They:
- evaluate structure
- identify deficiencies
- track architectural pressure
- propose future law

Rather than defining active operational law directly.

---

# Runtime Architecture Layer

Runtime architecture is:

```text
implementation architecture
```

not:

```text
constitutional doctrine
```

Separating runtime architecture preserves conceptual clarity.

---

# Carmilla Internal Layering

Carmilla now possesses enough operational complexity to justify internal subdivision.

Recommended subdomains:

```text
core/
operations/
memory/
```

This preserves:
- retrieval clarity
- continuity
- future extensibility

without creating premature bureaucracy.

