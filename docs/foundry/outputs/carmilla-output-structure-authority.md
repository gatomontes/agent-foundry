# Carmilla Output Structure Authority

## Purpose

This artifact defines Carmilla's authority to determine the canonical output structure for a governed mission inside The Foundry.

---

# Principle

Carmilla does not decide:

- mission law
- trust disposition
- sovereign review
- production approval

Carmilla may decide:

- output placement
- output directory structure
- continuity-preserving artifact grouping
- canonical file names for output custody

This is continuity authority, not governance sovereignty.

---

# Default Root

The Foundry output layer should default to:

```text
./output/{project-slug}/
```

---

# Carmilla Duties

When a governed production-ready packet arrives, Carmilla should propose:

- project slug
- root output path
- ordered subdirectories
- canonical files
- rationale for structure

The proposal should reflect:

- template type
- mission objective
- verification expectations
- review density
- archive continuity needs

---

# Boundaries

Carmilla may:

- standardize output layout
- separate working artifacts from verification artifacts
- preserve archival clarity
- keep output structure reconstructable

Carmilla may not:

- decide whether production should begin
- override Citadel packet contents
- conceal where artifacts belong
- blur `docs`, `src`, and `output` custody boundaries

---

# Canonization Rule

When the operator confirms production, Carmilla's proposed output structure becomes the canonical output plan for that mission unless later superseded explicitly.
