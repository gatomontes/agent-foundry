# The Foundry — Personnel Architecture

## Core Premise

The Foundry is not:
- a chatbot
- a single agent
- a monolithic prompt
- a persona

The Foundry is:
> an operational civilization engine.

It exists to transform ambiguous human intent into structured operational topology.

---

# Civilization Layers

```text
Doctrine
    ↓
Professions
    ↓
Personas
```

---

# I. Doctrine Layer

Doctrine defines the laws of operation.

This layer governs:
- delegation
- reporting
- escalation
- evidence handling
- uncertainty management
- critique
- execution semantics
- operational contracts

Examples:
- AFW
- Blackquill doctrine
- Observer protocol
- Subagent protocol
- Evidence standards
- Strategic operator doctrine

Doctrine is civilization law.

---

# II. Profession Layer

Professions are reusable operational archetypes.

A profession is NOT:
- a personality
- a character
- a mascot
- a persona

A profession IS:
- a capability mold
- a behavioral contract
- an operational specialization

Each profession defines:
- responsibilities
- operational boundaries
- workflows
- expected outputs
- escalation rules
- collaboration semantics
- evaluation criteria

Professions become the skeleton upon which personas are constructed.

---

# III. Persona Layer

Personas are expressions assembled from professions.

Example:

```text
Persona:
Carmilla

Composed From:
- Correspondence Steward
- Knowledge Architect
- Repository Steward
- Observer

Temperament:
calm
composed
gothic
minimalistic
```

The profession layer provides:
- capability
- structure
- workflow
- operational logic

The persona layer provides:
- identity
- temperament
- speech
- behavioral continuity
- symbolic atmosphere
