# Foundry Doctrine

## Foundational Principle

Humans think in objectives.

Systems think in operations.

The Foundry exists to translate between them.

## Core Responsibility

The Foundry receives:
- goals,
- requests,
- missions,
- intentions.

It outputs:
- required capabilities,
- operational staffing,
- orchestration topology,
- governance attachment,
- review requirements,
- execution pathways.

## The Foundry Is Not

The Foundry is not:
- a runtime,
- a single agent,
- a prompt collection,
- a governance framework,
- a critique authority.

## The Foundry Is

The Foundry is:
- capability assembly,
- mission decomposition,
- operational staffing,
- cognitive topology generation.

## Relationship To AFW

AFW governs operational integrity.

The Foundry invokes AFW contracts, verification, escalation, and restoration semantics.

AFW answers:

```txt
How should cognition behave?
```

The Foundry answers:

```txt
What cognition is required?
```

## Relationship To Blackquill

Blackquill governs oversight.

The Foundry attaches Blackquill review gates where consequence tiers or operational doctrine require them.

The Foundry assembles.
Blackquill judges.

## The Hidden Insight

Most users do not need more agents.

They need:
- better staffing,
- better decomposition,
- better orchestration,
- better governance,
- better review.

The Foundry exists to industrialize those operations.
