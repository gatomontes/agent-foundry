# Orchestration Flow — Rapid Prototype

## Purpose

This orchestration flow defines a lightweight operational pattern optimized for:

- exploration
- experimentation
- rapid iteration
- concept validation
- low-consequence implementation

The flow prioritizes learning velocity while preserving minimum governance discipline.

---

# Recommended Use Cases

Examples:

- proof-of-concept systems
- interface experimentation
- early-stage feature exploration
- workflow ideation
- gameplay or interaction prototyping
- orchestration experimentation

---

# Recommended Profession Topology

## Minimal Formation

- Systems Architect
- Runtime Operator
- Verification Specialist

## Optional Additions

Add only when justified:

- Critique Authority
- Executive Secretary

Avoid unnecessary orchestration density.

---

# Flow Topology

```txt
Executive Intake
  -> Rapid Decomposition
    -> Focused Execution
      -> Lightweight Verification
        -> Iterative Review
          -> Prototype Disposition
```

---

# Operational Characteristics

This flow favors:

- rapid iteration
- low coordination overhead
- contained experimentation
- fast learning cycles
- minimal staffing inflation

This flow intentionally sacrifices:

- exhaustive verification
- deep audit chains
- maximum governance friction

---

# Verification Expectations

Verification remains mandatory.

However:

- lightweight verification is acceptable
- audit may remain unnecessary
- critique may remain selective

Rapid iteration does not justify silent failure.

---

# Escalation Conditions

Escalate when:

- consequence tier increases
- persistence impact expands
- prototype becomes operational candidate
- governance exposure rises
- orchestration complexity increases

Prototype flows should not silently evolve into critical systems.

---

# Transition Rule

When operational significance increases:

Transition into:

- Verification Heavy Flow
- Structured SaaS Build Flow
- Governance-Aware Operational Flow

Governance pressure must scale with consequence.

---

# Common Failure Modes

- permanent prototypes
- hidden operationalization
- verification neglect
- architecture drift
- orchestration shortcuts
- underestimated persistence impact

---

# Constitutional Principle

Rapid experimentation is valuable only when the system remains honest about its provisional nature.
