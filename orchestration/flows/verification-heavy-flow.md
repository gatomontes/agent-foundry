# Orchestration Flow — Verification Heavy

## Purpose

This orchestration flow defines a governance-intensive operational pattern optimized for:

- high trust sensitivity
- elevated consequence tiers
- architectural significance
- financial correctness
- security exposure
- institutional continuity

The flow prioritizes trustworthy operational disposition over execution speed.

---

# Recommended Use Cases

Examples:

- payroll calculations
- security remediation
- governance redesign
- financial systems
- autonomous execution permissions
- constitutional doctrine updates
- critical SaaS infrastructure

---

## Core Chain

- Systems Architect
- Runtime Operator
- Verification Specialist
- Critique Authority
- Auditor

## Optional Coordination Layer

Add when orchestration density increases:

- Executive Secretary

# Flow Topology

```txt
Executive Intake
  -> Architecture Review
    -> Controlled Execution
      -> Independent Verification
        -> Critique Pressure Test
          -> Audit Review
            -> Coherence Validation
              -> Final Disposition
```

---

# Operational Characteristics

This flow favors:

- verification depth
- operational traceability
- escalation visibility
- restoration readiness
- artifact lineage preservation
- independent review

This flow intentionally sacrifices:

- raw execution speed
- orchestration minimality
- rapid iteration throughput

---

# Verification Requirements

Verification should favor:

- independent reasoning
- separate runtime pathways
- reproducible evidence
- explicit confidence reporting
- contradiction surfacing

Critical operations should not rely on self-verification.

---

# Escalation Behavior

Escalation sensitivity remains elevated.

Escalate when:

- contradiction remains unresolved
- verification confidence drops
- artifact lineage becomes ambiguous
- governance boundaries become unclear
- restoration becomes necessary

---

# Restoration Expectations

This flow assumes:

- failures will occur
- coherence may degrade
- restoration pathways must exist before continuation resumes

Operational resilience is mandatory.

---

# Common Failure Modes

- verification theater
- excessive trust in execution success
- audit bypass
- hidden governance drift
- suppressed escalation
- restoration neglect

---

# Constitutional Principle

High-consequence systems earn trust through disciplined operational friction.
